Audio Processing & NormalizationBefore the deep-learning classifier processes any incoming sample, the raw signal undergoes peak amplitude normalization:$$x_{\text{norm}}[n] = \frac{x[n]}{\max_{k} |x[k]|}$$This step scales the input range to $[-1, 1]$, mitigating structural volume variance across different microphonics.2. Feature Extraction (120 dimensions)The backend decodes each $3$-second audio clip and calculates a high-density, multi-dimensional audio array representing standard vocal tract attributes:┌──────────────────────────────────────────────────────────┐
│              120 Acoustic Feature Matrix                 │
├─────────────────────────┬────────────────────────────────┤
│ 40 Base MFCCs           │ Captures physical tract shape  │
├─────────────────────────┼────────────────────────────────┤
│ 40 Delta MFCCs          │ Velocity of frame deviations   │
├─────────────────────────┼────────────────────────────────┤
│ 40 Delta-Delta MFCCs    │ Acceleration rate of dynamics  │
└─────────────────────────┴────────────────────────────────┘
The resulting sequence matrix is reshaped to shape $(130, 120)$ and structured as a $(1, 130, 120)$ tensor, preparing it for spatial and temporal classification.3. Classification & Decision BoundaryThe input tensor flows through a hybrid 1D-Convolution and Recurrent Long Short-Term Memory (CNN-LSTM) network. The activation layer yields a prediction value $p \in [0.0, 1.0]$.$$\text{Decision} = \begin{cases} \text{Stuttering Detected} & \text{if } p \ge \text{Threshold} \\ \text{Fluent} & \text{if } p < \text{Threshold} \end{cases}$$[!IMPORTANT]The default decision boundary is established at $\text{Threshold} = 0.70$. This ensures high specificity and protects against background noise artifacts.📊 Dataset & TrainingThe classifier weights were trained on the industry-standard SEP-28k Dataset containing annotations from stuttering podcasts.Clip Collection: Generated over $20,000$ audio samples spanning diverse speakers.Feature Extraction: Pre-compiled into static structural files using $120$ MFCC arrays.Training Optimization: Backpropagated with balanced loss functions to counter class representation biases (majority of normal speech samples vs. stuttering intervals).🗂️ Project StructureFile / FolderRole in Pipelineapp.pyIntegrates FastAPI backend with the HTML5/JS client UI.stutter_detector_model.h5The compiled, ready-to-run Keras classification neural net.requirements.txtHolds dependency lists to spin up the program in virtual spaces.README.mdComprehensive developer documentation and runtime details.🚀 Getting StartedFollow these instructions to configure and run the software locally.PrerequisitesPython: Version 3.8 to 3.12 recommended.Pip: Standard Python package manager.Weights: Your pre-trained file stutter_detector_model.h5 placed in the root project directory.InstallationClone the Repository:git clone [https://github.com/yourusername/stuttering-detection-ai.git](https://github.com/yourusername/stuttering-detection-ai.git)
